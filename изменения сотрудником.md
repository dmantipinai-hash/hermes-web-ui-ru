# Изменения сотрудником

Дата: 05.06.2026

## Что изменено

### 1. Исправлен перенос карточек в `Входящие` / `Сделать`
- Найдена и исправлена ошибка, из-за которой UI писал, что задача перемещена, но фактически статус в базе не менялся.
- Исправлена серверная логика обновления статуса через SQLite.
- Исправлен путь к kanban DB для board-режима.
- При возврате задач назад дополнительно сбрасываются служебные поля (`current_run_id`, `claim_lock`, `claim_expires`, `worker_pid`, частично `completed_at`).

### 2. Разделены UX-колонки `Сделать` и `Готово к исполнению`
- Раньше `todo` и `ready` были визуально склеены в одну колонку.
- Теперь разделено:
  - `Входящие`
  - `Сделать`
  - `Готово к исполнению`
  - `Агент работает`
  - `Ждёт меня`
  - `Готово`
  - `Архив`
- Обновлены UI-маппинги и подписи колонок.
- Исправлен побочный баг, где `archived` местами визуально попадал в `done`.

### 3. Введён ручной workflow для совместной доски человек + агент
- Убрана зависимость `Входящих` от auto-triage семантики Hermes.
- Новые пользовательские задачи теперь создаются не как `triage`, а как припаркованные задачи с ручным управлением.
- При создании новой карточки теперь сразу записывается Web UI meta:
  - `ui_status: inbox`
  - `turn: user`
- Это нужно для того, чтобы новая задача оставалась во `Входящих` и не уходила автоматически в AI-спецификацию.

### 4. Новая логика колонок
- `Входящие` = ручная входящая корзина
- `Сделать` = ручная подготовка
- `Готово к исполнению` = очередь на исполнение
- `Агент работает` = реально исполняется агентом
- `Ждёт меня` = нужно действие пользователя
- `Готово` = завершено
- `Архив` = архив

### 5. Важное правило по исполнителю
- Если задача находится в `Готово к исполнению`, но исполнитель = пользователь,
  агент должен такую задачу игнорировать.
- Если исполнитель = агент, задача может идти в агентский поток исполнения.

### 6. Поведение кнопки «Отправить агенту»
- Теперь переводит задачу в:
  - backend status: `ready`
  - `turn: agent`
  - `ui_status: ready`
- То есть это теперь осмысленный перевод в агентскую очередь, а не просто смена turn.

6. Не появилось ли расхождение между Web UI meta и backend Hermes status.

## 06.06.2026

### Дополнительно внесённые изменения

1. Убрана верхняя плашка с предупреждением про Node.js 23.
- Плашка мешала интерфейсу и перекрывала верхнюю строку меню.
- Обновление Node не навязывается.

2. Убрана скрытая автоподстановка исполнителя при создании задачи.
- Если исполнитель не выбран вручную, задача больше не назначается автоматически на `default`.
- Это было одной из главных причин, почему карточки воспринимались как агентские и уходили в работу.

3. В форму создания задачи добавлены корректные варианты исполнителя.
- Теперь доступны:
  - `Без исполнителя`
  - `Я / пользователь`
  - агентские профили
- Это устраняет прежний корень проблемы, когда в UI фактически не существовало маршрута для человеческой задачи.

4. Добавлен псевдо-исполнитель `user`.
- Сервер теперь понимает `user` как допустимый ручной тип исполнителя.
- Он появляется в списках исполнителей и не должен трактоваться как агентский профиль.

5. Усилена логика интерпретации колонки `Готово к исполнению`.
- Если исполнитель пустой — задача не считается агентской.
- Если исполнитель `user` — задача не считается агентской.
- Только явно агентский исполнитель означает, что задача действительно может быть взята агентом.

### Выявленная логика / причина проблем

Главный корень найден:
- при создании задачи в UI не было нормального выбора человеческого маршрута;
- по сути существовал только агентский путь (`default` / агентский профиль),
  из-за чего карточка после создания начинала жить как агентская.

Именно это объясняет наблюдаемое поведение:
- карточка падала во `Входящие`,
- затем уходила дальше,
- затем Hermes/dispatcher считал, что задача уже может идти в агентскую обработку.

### Что проверить вручную после этого изменения

1. Новая задача с `Без исполнителя`:
- остаётся во `Входящих`;
- не уходит сама в `Готово к исполнению`;
- не попадает в `Агент работает`.

2. Новая задача с исполнителем `Я / пользователь`:
- остаётся человеческой задачей;
- агент её игнорирует.

3. Новая задача с агентским исполнителем:
- может быть вручную подготовлена и затем отправлена агенту.

4. Проверить, что перенос карточек мышкой между колонками теперь работает последовательно с новой логикой исполнителя.
